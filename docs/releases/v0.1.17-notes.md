# ao2-control-plane v0.1.17 release notes

**Release status:** candidate pending final qualification and publication.
**Target tag:** `v0.1.17`

## Patch scope

- Fixes native Windows rejected-smoke audit-log rotation behavior by leaving
  burst headroom under the 1 MiB hard audit-log cap.
- Keeps the newest rejected-smoke audit record during rotation and preserves
  well-formed JSONL evidence.
- Bounds the Windows burst test client's simultaneous TCP connects so release
  qualification exercises audit-log behavior instead of localhost accept-queue
  saturation.
- Preserves the Control Plane trust boundary: read-only observer surfaces only,
  no AO2 release approval, no provider execution, and no GitHub release mutation.

## Archive checksums

Archive checksums are generated from the exact `SHA256SUMS` manifest after the
candidate archives are built. Do not edit archive hashes by hand.
