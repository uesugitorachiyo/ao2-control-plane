## What's new

### `GET /api/v1/audit-log` operator endpoint
Bounded in-memory ring buffer of recent HTTP requests, exposed as JSON
through a new token-gated endpoint. Sibling to the structured
`tracing::info!(target: "ao2_cp_server::access")` access log — gives
operators a queryable view of the most recent N requests over HTTPS
without having to grep stderr on a remote host.

- Default capacity 1024 entries (~200 KiB resident); override via the
  new `AO2_CP_AUDIT_LOG_CAPACITY` environment variable. Appends past
  capacity evict the oldest entry (O(1) amortized).
- Bearer-token bytes are **never** copied into entries. Only the
  presence of an `Authorization` header (`auth_attempted`) and the
  outcome (`authenticated`) are recorded. The literal token can
  therefore never appear in the audit-log response or in the
  structured access-log tracing event.
- Filters compose: `limit` (clamped to `[1, 1024]`),
  `since_unix_micros`, `method` (case-insensitive), `status`,
  `status_class` (`"2xx"`…`"5xx"`), `path_prefix`, `authenticated`.
  Results are returned newest-first.

Response shape (`schema_version: "ao2.cp-audit-log.v1"`):

```json
{
  "schema_version": "ao2.cp-audit-log.v1",
  "buffer":   { "capacity": 1024, "total_buffered": 42 },
  "filtered_total": 7,
  "returned": 7,
  "limit": 100,
  "entries": [
    {
      "timestamp_unix_micros": 1764150000123456,
      "method": "GET",
      "path": "/api/v1/acceptance",
      "status": 200,
      "duration_micros": 421,
      "auth_attempted": true,
      "authenticated": true
    }
  ]
}
```

New route surface:

- `GET /api/v1/audit-log` — bearer-protected like every other
  `/api/v1/*` route. Categorised as `"audit"` in the route-catalog
  enum lane (`tests/release_packaging.rs::allowed_categories`),
  owner `"ao2-control-plane observer"`.

## Upgrade guide

- Binaries are drop-in replacements; on-disk data layout is
  unchanged from v0.1.3 (no migration required).
- The new endpoint is additive — existing clients are unaffected.
- The new `AO2_CP_AUDIT_LOG_CAPACITY` env var defaults to `1024`
  when unset, so existing deployments need no configuration change.

## Security posture (unchanged)

- OAuth-only authentication; no API-key provider auth.
- Bearer-token middleware unchanged; the audit-log route inherits
  the same `/api/v1/*` bearer gate. Tested explicitly via
  `audit_log_requires_bearer_token`.
- The audit-log endpoint does **not** widen the trust boundary:
  it surfaces only the same metadata already visible in the
  structured tracing access log (method, path, status, duration,
  auth-outcome). Bodies and `Authorization` header values are never
  recorded. Verified end-to-end by
  `audit_log_never_leaks_bearer_token_in_response`.
- Memory use is strictly capped by the ring-buffer capacity; no
  malicious-client path to unbounded growth.

## Cross-OS verification

All commits in this release are verified on macOS aarch64, Linux
x86_64, and Windows x86_64 via `scripts/smoke-three-os.sh`. New
regression coverage:

- `crates/ao2-cp-server/src/audit_log.rs` — 3 unit tests on the
  ring buffer (capacity eviction, zero-capacity opt-out, default).
- `crates/ao2-cp-server/tests/audit_log_endpoint.rs` — 7 integration
  tests covering auth, leak-resistance, filter composition, bounded
  eviction with capacity override, limit clamping (zero → 1,
  oversized → 1024), and `since_unix_micros` boundary filtering.
